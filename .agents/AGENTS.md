## 사전 승인 의무화 (절대 규칙 - Strict Execution Constraint)
1. **보고 우선:** 어떠한 경우라도 코드를 즉시 수정하지 않습니다. 수정, 삭제, 명령어 실행 전 반드시 "어떤 파일을 어떻게 바꿀 것인지" 구체적으로 보고합니다.
2. **작업 대기:** 계획 보고 후 즉시 턴을 종료하고 대기합니다. 
3. **승인 후 실행:** 사용자의 명시적인 승인(OK, 진행해 등)이 떨어진 이후에만 실제 도구(`write_to_file`, `run_command` 등)를 사용하여 작업을 실행합니다. (간단한 버그 수정이나 오타 수정 시에도 예외 없이 적용됩니다.)
